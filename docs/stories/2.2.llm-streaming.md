# Story 2.2: LLM Primitive (Streaming Mode)

## Status

Approved

## Story

**As a** Python developer using Beddel,
**I want** the `llm` primitive to support streaming mode via the `stream: true` config flag,
**so that** I can receive real-time token-by-token LLM responses as an async iterator instead of waiting for the full completion.

## Acceptance Criteria

1. When `config["stream"]` is `true`, the `llm_primitive` calls `provider.stream(request)` instead of `provider.complete(request)` and returns an `AsyncIterator[str]`
2. The `_build_request()` helper passes the `stream` field from config to `LLMRequest` (sets `stream=True` on the request model)
3. When `config["stream"]` is `false` or absent, the existing blocking behavior is preserved — `provider.complete(request)` is called and an `LLMResponse` is returned
4. Raise `PrimitiveError` with `BEDDEL-EXEC-001` when `llm_provider` is missing from context metadata (same as blocking mode)
5. Raise `ProviderError` with `BEDDEL-PROVIDER-001` when `provider.stream()` raises an exception, wrapping the original exception
6. The `stream` field in `LLMRequest` is set to `True` when streaming mode is active
7. Unit tests achieve >80% coverage for the streaming code paths in `primitives/llm.py`

### Previous Story Insights (from E2.1)

Story 2.1 implemented the blocking `llm_primitive` in `src/beddel-py/src/beddel/primitives/llm.py`. Key implementation details:

- **`_build_request(config)`** builds an `LLMRequest` from the resolved config dict. Currently extracts `model`, `messages`, `temperature`, `max_tokens`, `response_format` but does NOT extract `stream`. This needs to be extended.
- **`llm_primitive(config, context)`** extracts the provider from `context.metadata["llm_provider"]`, builds the request, calls `provider.complete(request)`, and returns the `LLMResponse`.
- **Error handling**: `PrimitiveError(BEDDEL-EXEC-001)` for missing provider, `ProviderError(BEDDEL-PROVIDER-001)` for provider failures with `raise ... from exc` chaining.
- **Registration**: `register_builtins()` in `primitives/__init__.py` registers `llm_primitive` under the name `"llm"`. No changes needed here — the same function handles both modes.
- **9 existing tests** in `tests/unit/primitives/test_llm.py` cover blocking mode. New streaming tests should be added to the same file.
- **Logger**: `logging.getLogger("beddel.primitives.llm")` — reuse for streaming debug logs.

### Data Models

The streaming feature leverages existing models from `beddel.domain.models`:

**`LLMRequest`** — already has the `stream` field:
```python
class LLMRequest(BaseModel):
    model: str
    messages: list[Message]
    temperature: float = 0.7
    max_tokens: int | None = None
    response_format: dict[str, Any] | None = None
    stream: bool = False  # ← KEY FIELD — set to True for streaming
```

**Return type changes**: When `stream=True`, the primitive returns `AsyncIterator[str]` (string chunks) instead of `LLMResponse`. The function signature becomes:
```python
async def llm_primitive(config, context) -> LLMResponse | AsyncIterator[str]
```

**No new models needed** — `LLMRequest.stream` and `AsyncIterator[str]` are already defined.

[Source: src/beddel-py/src/beddel/domain/models.py]

### ILLMProvider Port Interface

```python
@runtime_checkable
class ILLMProvider(Protocol):
    async def complete(self, request: LLMRequest) -> LLMResponse: ...
    async def stream(self, request: LLMRequest) -> AsyncIterator[str]: ...
```

The `stream()` method is already defined on the port. It accepts the same `LLMRequest` and returns `AsyncIterator[str]` — each yielded string is a text chunk from the LLM.

Note: `AsyncIterator` is imported from `collections.abc` under `TYPE_CHECKING` in ports.py.

[Source: src/beddel-py/src/beddel/domain/ports.py]

### Implementation Pattern

The streaming branch follows the same structure as the blocking path:

```python
async def llm_primitive(config, context) -> LLMResponse | AsyncIterator[str]:
    provider = context.metadata.get("llm_provider")
    if provider is None:
        raise PrimitiveError(...)  # Same as before

    request = _build_request(config)

    if request.stream:
        try:
            return await provider.stream(request)  # Returns AsyncIterator[str]
        except Exception as exc:
            raise ProviderError(...) from exc

    # Existing blocking path unchanged
    try:
        response = await provider.complete(request)
    except Exception as exc:
        raise ProviderError(...) from exc
    return response
```

Key design decision: The `provider.stream()` method is `async` and returns an `AsyncIterator[str]`. The primitive `await`s the call to get the iterator, then returns it directly. The caller (executor or framework adapter) is responsible for consuming the iterator.

[Source: src/beddel-py/src/beddel/domain/ports.py, docs/architecture/components.md]

### Testing Pattern for AsyncIterator

To mock `provider.stream()`, create an async generator and configure the mock:

```python
async def mock_stream_chunks():
    for chunk in ["Hello", " ", "world"]:
        yield chunk

mock_provider.stream = AsyncMock(return_value=mock_stream_chunks())
```

To verify the returned iterator, consume it in the test:

```python
result = await llm_primitive(config, context)
chunks = [chunk async for chunk in result]
assert chunks == ["Hello", " ", "world"]
```

[Source: pytest-asyncio documentation, existing test patterns in test_llm.py]

### YAML Config Shape (streaming)

```yaml
- id: "stream-greet"
  type: "llm"
  config:
    model: "openrouter/anthropic/claude-3"
    system: "You are a helpful assistant."
    messages:
      - role: "user"
        content: "$input.prompt"
    stream: true  # ← Activates streaming mode
  result: "greeting"
```

```python
{
    "model": "openrouter/anthropic/claude-3",
    "system": "You are a helpful assistant.",
    "messages": [{"role": "user", "content": "Hello world"}],
```
