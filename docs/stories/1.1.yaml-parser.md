# Story 1.1: YAML Parser with Pydantic Validation

## Status

Done

## Story

**As a** Python developer using Beddel,
**I want** to parse YAML workflow definitions into validated Pydantic models,
**so that** I can define AI pipelines declaratively with confidence that my workflow files are structurally correct before execution.

## Acceptance Criteria

1. Parse YAML workflow strings into `WorkflowDefinition` Pydantic models using `yaml.safe_load()` exclusively (NEVER `yaml.load()`)
2. Parse YAML workflow files from disk via `parse_file(path)` with proper file-not-found error handling
3. Validate all required fields via Pydantic: `metadata` (name, version), `workflow` (list of steps with id, type, config)
4. Support optional fields: `config` (global workflow config with defaults), `return` template (aliased to `return_template`), step-level `result`, `condition`, `on_error`
5. Perform semantic validation beyond Pydantic schema: duplicate step IDs, max_steps exceeded
6. Raise `ParseError` with structured error codes (`BEDDEL-PARSE-001` for invalid YAML, `BEDDEL-PARSE-002` for validation failures, `BEDDEL-PARSE-003` for file not found)
7. All existing YAML fixtures (`simple.yaml`, `multi_step.yaml`, `nested_agent.yaml`) parse successfully
8. Unit tests achieve >80% coverage for `parser.py` and related models in `models.py`

### Data Models

All models are defined in `src/beddel-py/src/beddel/domain/models.py`:

- `WorkflowDefinition`: Root model with `metadata`, `workflow`, `config` (default), `return_template` (alias `return`)
- `StepDefinition`: `id`, `type`, `config` (dict), `result`, `condition`, `on_error` — all optional fields have defaults
- `WorkflowMetadata`: `name` (required), `version` (default "1.0.0"), `description` (optional)
- `WorkflowConfig`: `timeout_seconds=300`, `max_steps=50`, `environment={}` 
- `ErrorHandler`: `strategy="fail"` (fail | skip)
- Error hierarchy: `BeddelError(Exception)` → `ParseError`, `ResolutionError`, `ExecutionError` → `PrimitiveError`, `ProviderError`; plus `ConfigurationError`
- `ErrorCode(StrEnum)`: `BEDDEL-PARSE-001` (invalid YAML), `BEDDEL-PARSE-002` (validation), `BEDDEL-PARSE-003` (file not found)

[Source: docs/architecture/data-models.md, existing models.py]

### Component Specifications

`YAMLParser` component from architecture:
- `parse(yaml_content: str) -> WorkflowDefinition` — Parse YAML string
- `parse_file(path: Path) -> WorkflowDefinition` — Parse from file
- `validate(definition: WorkflowDefinition) -> list[str]` — Semantic validation
- Dependencies: Pydantic, PyYAML
- Security: ONLY `yaml.safe_load()`, NEVER `yaml.load()`
