# Story 3.1: LiteLLM Adapter (Unified Provider Interface)

## Status

Done

## Story

**As a** Python developer using Beddel,
**I want** a LiteLLM adapter that implements the `ILLMProvider` port,
**so that** my workflows can call any LLM provider (OpenRouter, Gemini, Bedrock, etc.) through a unified async interface without changing workflow definitions.

## Acceptance Criteria

1. Create a `LiteLLMAdapter` class in `src/beddel-py/src/beddel/adapters/litellm.py` that implements the `ILLMProvider` protocol
2. Implement `async def complete(self, request: LLMRequest) -> LLMResponse` that calls `litellm.acompletion()` and maps the response to an `LLMResponse` model
3. Implement `async def stream(self, request: LLMRequest) -> AsyncIterator[str]` that calls `litellm.acompletion(stream=True)` and yields content chunks as strings
4. Map `LLMRequest` fields to LiteLLM parameters: `model`, `messages` (as list of dicts), `temperature`, `max_tokens`, `response_format`
5. Map LiteLLM response to `LLMResponse`: extract `content`, `model`, `usage` (prompt/completion/total tokens), `finish_reason`
6. Raise `ProviderError` with `BEDDEL-PROVIDER-001` when LiteLLM raises any exception during `complete()` or `stream()`
7. Support optional `api_base` and `api_key` constructor parameters for provider-specific configuration (passed through to LiteLLM calls)
8. Log provider calls at DEBUG level via structured logging (model, token usage on success, error on failure)
9. Verify the adapter satisfies `isinstance(adapter, ILLMProvider)` at runtime (Protocol with `runtime_checkable`)
10. Unit tests achieve >80% coverage for `adapters/litellm.py`, mocking `litellm.acompletion` to avoid real API calls

### ILLMProvider Port Interface

The adapter must implement this Protocol (already defined in `domain/ports.py`):

```python
@runtime_checkable
class ILLMProvider(Protocol):
    async def complete(self, request: LLMRequest) -> LLMResponse: ...
    async def stream(self, request: LLMRequest) -> AsyncIterator[str]: ...
```

[Source: src/beddel-py/src/beddel/domain/ports.py]

### LLMRequest → LiteLLM Parameter Mapping

| LLMRequest field | LiteLLM param | Notes |
|------------------|---------------|-------|
| `model` | `model` | Direct pass-through (e.g., `"openrouter/anthropic/claude-3"`) |
| `messages` | `messages` | Convert `Message` objects to `[{"role": ..., "content": ...}]` dicts |
| `temperature` | `temperature` | Direct pass-through |
| `max_tokens` | `max_tokens` | Direct pass-through, skip if `None` |
| `response_format` | `response_format` | Direct pass-through for structured outputs (JSON Schema dict) |
| `stream` | `stream` | Set by method: `False` for `complete()`, `True` for `stream()` |

Constructor params `api_key` and `api_base` are passed as additional kwargs to every `litellm.acompletion()` call when set.

[Source: docs/architecture/external-apis.md, docs/architecture/data-models.md]

### LiteLLM Response → LLMResponse Mapping

```python
# litellm.acompletion() returns a ModelResponse with:
response.choices[0].message.content  → LLMResponse.content
response.model                       → LLMResponse.model
response.usage.prompt_tokens         → TokenUsage.prompt_tokens
response.usage.completion_tokens     → TokenUsage.completion_tokens
response.usage.total_tokens          → TokenUsage.total_tokens
response.choices[0].finish_reason    → LLMResponse.finish_reason
```

For streaming, `litellm.acompletion(stream=True)` returns an async iterator of `ModelResponseStream` chunks:
```python
chunk.choices[0].delta.content  # str | None — yield when not None
```

[Source: https://docs.litellm.ai/ — LiteLLM API reference]

### Data Models Used

`LLMRequest` (from `beddel.domain.models`):
- `model: str` — Model identifier
- `messages: list[Message]` — Conversation messages (role + content)
- `temperature: float = 0.7`
- `max_tokens: int | None = None`
- `response_format: dict[str, Any] | None = None`
- `stream: bool = False`

`LLMResponse` (from `beddel.domain.models`):
- `content: str` — Response text
- `model: str` — Actual model used
- `usage: TokenUsage` — Token counts (prompt, completion, total)
- `finish_reason: str = "stop"`
- `created_at: datetime` — Auto-set

`ProviderError(ExecutionError)` — for provider-specific failures:
- `ErrorCode.PROVIDER_ERROR` (`BEDDEL-PROVIDER-001`)
- `ErrorCode.PROVIDER_TIMEOUT` (`BEDDEL-PROVIDER-002`)

`Message`:
- `role: str` — system | user | assistant
- `content: str`

[Source: src/beddel-py/src/beddel/domain/models.py]

### How Primitives Consume the Provider

The `llm` and `chat` primitives retrieve the provider from context:
```python
provider: ILLMProvider | None = context.metadata.get("llm_provider")
```

They call `provider.complete(request)` or `provider.stream(request)`. The adapter is injected into `ExecutionContext.metadata["llm_provider"]` at engine creation time (E2E wiring in Phase 3).

[Source: src/beddel-py/src/beddel/primitives/llm.py, src/beddel-py/src/beddel/primitives/chat.py]

### Reviewed By: Quinn (Test Architect)

### Code Quality Assessment

**Overall: Excellent.** The LiteLLM adapter is a clean, well-structured implementation of the hexagonal adapter pattern. Key observations:

- **Architecture**: Correct hexagonal boundary — adapter depends inward on domain ports (`ILLMProvider`) and models (`LLMRequest`, `LLMResponse`, `TokenUsage`, `ProviderError`). No domain leakage outward. External dependency (`litellm`) is fully encapsulated.
- **Protocol compliance**: `LiteLLMAdapter` satisfies `ILLMProvider` via structural subtyping. `isinstance()` check passes at runtime thanks to `@runtime_checkable` on the Protocol and `from __future__ import annotations` enabling deferred evaluation.
- **Error handling**: All exceptions from `litellm.acompletion()` are caught and wrapped in `ProviderError` with `ErrorCode.PROVIDER_ERROR` (`BEDDEL-PROVIDER-001`). The `except ProviderError: raise` guard in `stream()` correctly prevents double-wrapping. Exception chaining via `from exc` preserves the original traceback.
- **Null safety**: Defensive fallbacks for `response.usage` (→ `litellm.Usage` zero-fill), `choice.message.content` (→ `""`), `response.model` (→ `request.model`), and `choice.finish_reason` (→ `"stop"`).
- **Logging**: Structured DEBUG logging via named logger `beddel.adapters.litellm` with %-style formatting (no f-strings in log calls). Logs on entry, success, and failure for both `complete()` and `stream()`.
- **Type hints**: Full type annotations on all public methods. `TYPE_CHECKING` guard correctly used for import-only types (`AsyncIterator`, `LLMRequest`).
