## Story

**As a** Python developer integrating Beddel with a web framework,
**I want** a dedicated, framework-agnostic SSE streaming adapter in `beddel.integrations.sse` that encapsulates Server-Sent Events protocol formatting, event serialization, and error handling,
**so that** I can stream real-time LLM responses through any ASGI-compatible framework (not just FastAPI) with consistent event formatting, proper backpressure handling, and clean separation of SSE concerns from HTTP handler logic.

## Acceptance Criteria

1. Create a `BeddelSSEAdapter` class in `src/beddel-py/src/beddel/integrations/sse.py` that encapsulates all SSE protocol logic (event formatting, serialization, error events, done sentinel)
2. The adapter accepts an `ExecutionResult` (or an `AsyncIterator[str]` directly) and produces a well-typed async iterator of `SSEEvent` dataclass instances with `event`, `data`, and optional `id`/`retry` fields
3. SSE events follow the established format from Story 5.1: `event: chunk` + `data: {text}` for content chunks, `event: done` + `data: [DONE]` for completion, `event: error` + `data: {error_json}` for errors
4. The adapter handles all Beddel exception types (`ParseError`, `ConfigurationError`, `ExecutionError`, `ProviderError`) and maps them to structured SSE error events with JSON payloads containing `code`, `message`, and `details`
5. Refactor `integrations/fastapi.py` to consume `BeddelSSEAdapter` instead of the inline `_sse_generator` â€” the FastAPI handler delegates SSE formatting entirely to the adapter
6. The `SSEEvent` dataclass provides a `serialize()` method that returns the raw SSE wire format string (`event: {event}\ndata: {data}\n\n`) for use by non-FastAPI frameworks
7. Export `BeddelSSEAdapter` and `SSEEvent` from `beddel.integrations.sse` (importable as `from beddel.integrations.sse import BeddelSSEAdapter, SSEEvent`)
8. The adapter is framework-agnostic â€” it has zero dependencies on FastAPI or sse-starlette (only stdlib + beddel domain types)
9. Unit tests achieve >80% coverage for `integrations/sse.py`
10. All existing Story 5.1 tests continue to pass after the refactor of `fastapi.py`

### Architecture Context

The SSE adapter sits in the `integrations/` layer, outside the hexagonal architecture boundary. It consumes domain types (`ExecutionResult`, `BeddelError` hierarchy) but has zero framework dependencies â€” it's a pure Python module that formats SSE events. The FastAPI handler (`fastapi.py`) then bridges between this adapter and the `sse-starlette` library.

This separation enables:
- Reuse with other ASGI frameworks (Starlette standalone, Litestar, etc.)
- Testing SSE logic without FastAPI/sse-starlette installed
- Clean single-responsibility: adapter formats events, handler manages HTTP

[Source: docs/architecture/components.md, docs/architecture/core-workflows.md]

### SSE Protocol Reference

Server-Sent Events wire format (per [HTML Living Standard](https://html.spec.whatwg.org/multipage/server-sent-events.html)):
```
event: chunk
data: Hello world
id: 1

event: done
data: [DONE]

```

Each event is terminated by a blank line (`\n\n`). Fields are `event:`, `data:`, `id:`, `retry:`. The `data:` field can span multiple lines (each prefixed with `data:`). The `id:` field sets the last event ID for reconnection. The `retry:` field sets the reconnection time in milliseconds.

### Security Review

- **No information leakage**: Error events use `str(exc)` which includes the BeddelError code prefix â€” this is structured and intentional. Generic exceptions expose their message via `INTERNAL_ERROR`, which is acceptable for development but should be reviewed before production deployment (consider sanitizing internal exception messages).
- **No credential exposure**: SSE adapter handles only domain types, no auth tokens or API keys flow through this layer.
- **No injection risk**: `json.dumps()` safely serializes error payloads â€” no raw string interpolation in wire format.

### Performance Considerations

- **Async generator overhead**: Minimal â€” one `SSEEvent` dataclass allocation per chunk. `frozen=True, slots=True` ensures low memory footprint.
- **No buffering**: Events are yielded immediately as they arrive â€” correct for streaming use cases.
- **Backpressure**: Delegated to the consuming framework (e.g., `EventSourceResponse` in FastAPI). The adapter itself does not implement backpressure, which is the correct design for a formatting layer.
- **Non-streaming path**: `json.dumps(result.model_dump(mode="json"))` serializes the full result â€” acceptable for non-streaming responses which are expected to be small.
