# Story 1.2: Variable Resolver

## Status

Done

## Story

**As a** Python developer using Beddel,
**I want** variable references like $input.*, $stepResult.*, and $env.* to be resolved recursively in workflow configurations,
**so that** I can build dynamic workflows where step outputs feed into subsequent steps without manual data wiring.

## Acceptance Criteria

1. Resolve `$input.*` references from the workflow input data
2. Resolve `$stepResult.*` references from previous step outputs (keyed by step's `result` field)
3. Resolve `$env.*` references from execution context env dict, falling back to `os.environ`
4. Support recursive resolution through nested dicts and lists
5. Full-string variable references (e.g., `"$input.data"`) preserve the original type (dict, list, int, etc.)
6. Inline variable references within strings (e.g., `"Hello $input.name"`) are substituted as strings
7. Raise `ResolutionError` with `BEDDEL-RESOLVE-001` for unknown variables and `BEDDEL-RESOLVE-002` for invalid paths
8. Dotted path traversal works for nested dict access (e.g., `$stepResult.analysis.text`)

## Dev Notes

### Previous Story Insights
Story 1.1 established the Pydantic models including ExecutionContext with with_step_result() immutability pattern, and the error hierarchy (BeddelError -> ResolutionError with ErrorCode enum).

### Existing Implementation State
VariableResolver is fully implemented in src/beddel-py/src/beddel/domain/resolver.py:
- Regex pattern: `$(?P<namespace>input|stepResult|env)\.(?P<path>[a-zA-Z0-9_.]+)`
- resolve(template, context) handles str, dict, list, and passthrough types
- resolve_dict(data, context) resolves all values in a dict recursively
- _resolve_string() distinguishes full-string match (type-preserving) from inline substitution
- _lookup() routes to input, stepResult, or env namespace
- _traverse() walks dotted paths through nested dicts with detailed error messages
- env resolution falls back to os.environ if key not in context.env

[Source: src/beddel-py/src/beddel/domain/resolver.py]

### Data Models
- ExecutionContext: workflow_id, input (dict), step_results (dict), env (dict), metadata (dict)
- ExecutionContext.with_step_result(key, result) returns new context (immutable pattern)
- ResolutionError(BeddelError) with codes BEDDEL-RESOLVE-001 (unknown var) and BEDDEL-RESOLVE-002 (invalid path)

[Source: docs/architecture/data-models.md, src/beddel-py/src/beddel/domain/models.py]
