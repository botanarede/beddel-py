# Story 1.4: Primitive Registry with Decorator Pattern

## Status

Done

## Story

**As a** Python developer extending Beddel,
**I want** a registry that manages primitive functions with decorator-based registration and lookup,
**so that** I can register custom primitives and the executor can dynamically resolve step types to their implementations at runtime.

## Acceptance Criteria

1. Register primitives via decorator pattern: `@registry.register("name")` on async functions
2. Register primitives imperatively via `register_func(name, fn)` for programmatic registration
3. Look up registered primitives by name via `get(name)` returning the async callable
4. Raise `ExecutionError` with `BEDDEL-EXEC-002` when looking up an unregistered primitive
5. Prevent duplicate registration — raise `ValueError` if name already registered
6. List all registered primitive names via `list()` (sorted alphabetically)
7. Check primitive existence via `has(name)` returning bool
8. Primitive functions follow the signature: `async (config: dict, context: ExecutionContext) -> Any`

Stories 1.1-1.3 established the domain models including ExecutionContext (used in PrimitiveFunc signature), ExecutionError with ErrorCode.EXEC_PRIMITIVE_NOT_FOUND, and the WorkflowExecutor which depends on PrimitiveRegistry for step type resolution.

### Existing Implementation State
PrimitiveRegistry is fully implemented in src/beddel-py/src/beddel/domain/registry.py:
- PrimitiveFunc type alias: Callable[[dict[str, Any], ExecutionContext], Awaitable[Any]]
- Internal storage: dict[str, PrimitiveFunc]
- register(name) decorator: validates no duplicates, stores function, returns it unchanged
- register_func(name, fn): imperative alternative to decorator
- get(name): returns function or raises ExecutionError(BEDDEL-EXEC-002) with available primitives in details
- list(): returns sorted(self._primitives.keys())
- has(name): returns name in self._primitives

[Source: src/beddel-py/src/beddel/domain/registry.py]

### Data Models
- PrimitiveFunc = Callable[[dict[str, Any], ExecutionContext], Awaitable[Any]]
- ExecutionError(BeddelError) with code=ErrorCode.EXEC_PRIMITIVE_NOT_FOUND
- Error details include: {"primitive": name, "available": list of registered names}

[Source: src/beddel-py/src/beddel/domain/models.py, registry.py]

### Component Specifications
PrimitiveRegistry from architecture:
- register(name: str) -> Callable[[PrimitiveFunc], PrimitiveFunc] (decorator)
- get(name: str) -> PrimitiveFunc
- list() -> list[str]
- Dependencies: None (standalone registry)
- Technology: Python decorator pattern with dict storage

[Source: docs/architecture/components.md#PrimitiveRegistry]

### Integration with Executor
The WorkflowExecutor calls registry.get(step.type) during step execution to resolve the primitive function. If not found, ExecutionError propagates and stops the workflow.

[Source: src/beddel-py/src/beddel/domain/executor.py]
