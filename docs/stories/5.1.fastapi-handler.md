# Story 5.1: createBeddelHandler FastAPI Factory

## Status

InProgress

## Story

**As a** Python developer building a web application with Beddel,
**I want** a `create_beddel_handler` factory function in `beddel.integrations.fastapi` that accepts a `WorkflowDefinition` (or YAML path) and returns a FastAPI-compatible async handler function,
**so that** I can expose my AI workflows as HTTP endpoints with minimal boilerplate, including SSE streaming support for real-time LLM responses.

## Acceptance Criteria

1. Create a `create_beddel_handler()` factory function in `src/beddel-py/src/beddel/integrations/fastapi.py` that returns an async FastAPI route handler
2. The factory accepts: `workflow` (WorkflowDefinition or str path to YAML), optional `provider` (ILLMProvider), optional `hooks` (list[ILifecycleHook]), optional `tracer` (ITracer), optional `registry` (PrimitiveRegistry)
3. The returned handler accepts a JSON request body (dict), executes the workflow via `WorkflowExecutor`, and returns the `ExecutionResult` as JSON
4. When the workflow contains a streaming step (`stream: true`), the handler returns an SSE `EventSourceResponse` instead of JSON — streaming chunks as `data:` events and a final `data: [DONE]` sentinel
5. SSE events follow the format: `event: chunk\ndata: {text}\n\n` for content chunks, `event: done\ndata: [DONE]\n\n` for completion, and `event: error\ndata: {error_json}\n\n` for errors
6. All Beddel exceptions (`BeddelError` subclasses) are caught and returned as structured JSON error responses with appropriate HTTP status codes (400 for `ParseError`/`ConfigurationError`, 500 for `ExecutionError`/`ProviderError`)
7. The factory function validates that FastAPI dependencies are available at import time — raises `ImportError` with a helpful message if `fastapi` or `sse-starlette` are not installed
8. Export `create_beddel_handler` from `beddel.integrations.fastapi` (importable as `from beddel.integrations.fastapi import create_beddel_handler`)
9. Unit tests achieve >80% coverage for `integrations/fastapi.py`
10. The handler sets appropriate CORS-friendly headers for SSE responses (`Cache-Control: no-cache`, `Connection: keep-alive`)

### WorkflowExecutor Interface

The executor is the main entry point for workflow execution:

```python
class WorkflowExecutor:
    def __init__(
        self,
        registry: PrimitiveRegistry,
        resolver: VariableResolver | None = None,
        tracer: ITracer | None = None,
        hooks: list[ILifecycleHook] | None = None,
    ) -> None: ...

    async def execute(
        self,
        workflow: WorkflowDefinition,
        input_data: dict[str, Any] | None = None,
    ) -> ExecutionResult: ...
```

The executor returns `ExecutionResult` which has `.model_dump()` for JSON serialization. For streaming steps, the step result's `.output` is an `AsyncIterator[str]` — the handler needs to detect this and convert to SSE events.

[Source: src/beddel-py/src/beddel/domain/executor.py]

### LLM Primitive Streaming

The `llm_primitive` returns `AsyncIterator[str]` when `config["stream"]` is `True`. The iterator yields content strings as they arrive from the provider. The handler needs to:
1. Detect that a step result is an `AsyncIterator`
2. Iterate over it, yielding SSE events for each chunk
3. Send `[DONE]` sentinel when the iterator is exhausted

[Source: src/beddel-py/src/beddel/primitives/llm.py]

### Error Handling

Beddel exception hierarchy for HTTP status mapping:
- `ParseError` (YAML/validation) → HTTP 400
- `ConfigurationError` (invalid config) → HTTP 400
- `ExecutionError` / `PrimitiveError` → HTTP 500
- `ProviderError` (LLM provider) → HTTP 500
- Generic `Exception` → HTTP 500

Error response format:
```json
{
    "error": {
        "code": "BEDDEL-PARSE-001",
        "message": "Invalid YAML: ...",
        "details": {}
    }
}
```

[Source: docs/architecture/error-handling-strategy.md]
